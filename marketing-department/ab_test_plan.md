# Протокол A/B тестирования — Операция «Геродот»

## 1. Цель тестирования
Определить наиболее эффективную комбинацию темы письма и первого абзаца для повышения:
- Open Rate (OR) — % открытий
- Click-Through Rate (CTR) — % переходов
- Reply Rate (RR) — % ответов

## 2. Гипотезы

### Гипотеза A: Юмор снижает барьер
Тема: «Строим обед за 30 секунд. Как у вас?»
Абзац: лёгкий, с иронией. Цель — повысить Reply Rate.

### Гипотеза B: Польза и выгода
Тема: «Как сэкономить 40% на корпоративном питании»
Абзац: цифры, экономия, ROI. Цель — повысить CTR.

### Гипотеза C: Социальное доказательство
Тема: «Почему Яндекс и Тинькофф выбрали Gastroprime»
Абзац: кейсы, имена, результаты. Цель — повысить Open Rate.

## 3. Методология

| Параметр | Значение |
|---|---|
| Метод сплитования | Случайное A/B/C, равные сегменты |
| Размер выборки | 300 контактов (по 100 на вариант) |
| Длительность теста | 7 дней |
| Уровень значимости | 95% (p < 0.05) |
| Доверительный интервал | ±5% |

## 4. Отслеживаемые метрики

- Open Rate (OR) = (Открытия / Доставлено) × 100
- Click Rate (CTR) = (Клики / Доставлено) × 100
- Reply Rate (RR) = (Ответы / Доставлено) × 100
- Bounce Rate = (Отказы / Отправлено) × 100
- Unsubscribe Rate = (Отписки / Доставлено) × 100

## 5. Критерии победы

- Вариант считается победителем при OR выше на ≥10% относительно baseline
- Если разница < 5% — объявляем ничью, комбинируем лучшие элементы
